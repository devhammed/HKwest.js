HKwest.js AJAX/HTTP Library

HKwest.js is a simple and minimalist Javascript Library for AJAX, HTTP Requests and CORS Requests.

Request Examples

HKwest.js supports these HTTP Methods (GET, POST, PUT, PATCH, DELETE, OPTIONS, TRACE, CONNECT) and can be use to develop AJAX and RESTful projects.
HKwest.js would return XMLHttpRequest/XDomainRequest object to your callback function after successful request so you will have access to all Javascript Native methods e.g responseText, responseXML, status, statusText, getAllResponseHeaders, getResponseHeader(header) etc.
Below is an example GET request to HTTPBIN.org site:
<script>
HKwest({
url: 'http://httpbin.org/get?q=name',
done: function (xhr) {
// do something with XHR object
// alert(xhr.responseText);
}});
</script>

Below is an example POST requests to HTTPBIN.org
<script>
HKwest({
url: 'http://httpbin.org/post',
type: 'POST',
data: {
"name": "Oyedele Hammed Horlah",
"age": 17,
"bio": "I'm a Self-taught Computer Programmer and Web Developer"
},
headers: {
"X-Key": "value",
"X-Name": "horlahcoded"
},
done: function (xhr) {
// do something with XHR object
// alert(xhr.responseText);
}});
</script>

Options and Settings

HKwest.js options are used to configure the request. some of this options are optional and some are required.

url : Request URL (required)

type : Request Method (optional, default: GET)

data : Request Body, it can be a String or Object (key/value) or JSON String or FormData Object (optional)

headers : Request Headers in Keys and Values Pairs (Object) (optional)

contentType : Request Body Data Type (default: application/www-form-url-encoded, optional)

auth : Request Authentication, username:password String e.g auth: 'username:password' (optional)

mime : Override Response Data Mime(optional)

done : Callback Function, HKwest would pass the XHR Object to this Function so that you would have access to these properties (response, responseText, responseXML, responseURL, responseType, status, statusText, getAllResponseHeaders(), getResponseHeader(header) and readyState). (optional, default: Console Log)

returnType : This would tell XHR the type of response that would be received, it accepts (text, arraybuffer, blob, json, document) values only. (default: text, optional)
